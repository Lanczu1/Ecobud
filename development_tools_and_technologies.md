# 3.X Development Tools and Technologies

This chapter presents the tools, frameworks, programming languages, libraries, and technologies used in the design and implementation of ECOBUD. Each technology is discussed according to its definition, its general function, its specific application within the system, and the rationale behind its selection. The technologies are organized according to the system architecture layers to which they belong.

---

## 3.X.1 Client Layer Technologies

### React Native with Expo

According to Eisenman (2015), React Native is an open-source user interface framework created by Meta Platforms that enables developers to build natively rendered mobile applications for Android and iOS using JavaScript and the React component model. Expo, as described by Expo (2024), is an open-source platform built around React Native that provides a managed workflow and a library of pre-built native modules, eliminating the need for direct native project configuration. React Native renders genuine native platform components rather than web-based views, while Expo provides ready-to-use device APIs for camera access, image handling, storage, gradient effects, and video playback. In ECOBUD, React Native with Expo version 56.0.15 is used to build the primary mobile application in `apps/mobile`, providing screens for eco-habit logging, challenge tracking, streak monitoring, lesson consumption, community event participation, badge collection, and gamification dashboard viewing. Supplementary Expo modules include `expo-image`, `expo-image-picker`, `expo-camera`, `expo-linear-gradient`, `expo-video`, and `expo-sqlite`. The developers selected React Native with Expo because its single shared TypeScript codebase eliminates the overhead of maintaining separate Android and iOS codebases, and Expo's managed workflow enables rapid testing through the Expo Go companion application. Therefore, the developers chose React Native with Expo as the mobile application framework for the development of ECOBUD.

---

### React

According to Banks and Porcello (2017), React is an open-source frontend JavaScript library developed by Meta Platforms for building user interfaces through a declarative, component-based architecture that efficiently updates and renders only the components affected by data changes using a virtual DOM reconciliation algorithm. React operates through a tree of reusable, stateful components in which each component manages its own rendering logic and receives data through a unidirectional data flow, making application state predictable and maintainable at scale. In ECOBUD, React version 19.2.4 is used for the `apps/web` application, which serves as the unified administrator and moderation dashboard. The dashboard consolidates interfaces for user management, learning content authoring, challenge configuration, community event management, swap goods moderation, report review, submission approval workflows, and audit log viewing. The developers selected React because its component-based architecture enables a high degree of code reuse across the numerous administrative modules, and its declarative rendering model simplifies the management of complex dashboard state. Therefore, the developers chose React as the frontend library for the web administrator dashboard of ECOBUD.

---

### Vite

According to You (2024), Vite is a next-generation frontend build tool that provides a fast local development server leveraging native ES module imports in the browser, and a pre-configured Rollup-based bundler for optimized production builds. Unlike traditional bundler-based development servers that must process the entire application before serving, Vite serves source files directly over native ESM during development, resulting in near-instant server startup and sub-second Hot Module Replacement regardless of application size. In ECOBUD, Vite version 8.0.4 serves as the build tool and development server for the `apps/web` application, running on port 5174 during local development. It is configured with the `@vitejs/plugin-react` plugin version 6.0.1 for React JSX transformation and Fast Refresh support, and the `@tailwindcss/vite` plugin version 4.2.2 for Tailwind CSS compilation. The developers selected Vite because its native ESM-based development server provides a significantly faster development experience than traditional alternatives, and its plugin architecture enabled seamless integration with both React and Tailwind CSS within a single configuration file. Therefore, the developers chose Vite as the frontend build tool and development server for the web platform of ECOBUD.

---

### Tailwind CSS

According to Wathan (2019), Tailwind CSS is a utility-first CSS framework that provides a comprehensive set of low-level utility classes applied directly in HTML or JSX markup, enabling developers to construct custom designs without writing separate CSS stylesheets or being constrained by pre-built component styles. Tailwind operates by scanning source files for utility class names and generating only the CSS required for those classes, resulting in minimal production stylesheet sizes. A class-based dark mode strategy allows the entire application theme to be toggled by adding or removing a single class on the root HTML element. In ECOBUD, Tailwind CSS version 4.2.2 is used for all styling across the `apps/web` administrator dashboard, providing responsive layouts, a dark mode theme, and a consistent visual design system across all administrative pages including the dashboard overview, user management table, learning content editor, challenge and event configuration forms, and audit log viewer. The developers selected Tailwind CSS because its utility-first approach eliminated the need to maintain a separate CSS codebase, enabling rapid UI iteration while keeping styles co-located with the components they affect. Therefore, the developers chose Tailwind CSS as the styling framework for the web administrator dashboard of ECOBUD.

## 3.X.2 API Layer Technologies

### Node.js

According to Cantelon et al. (2014), Node.js is an open-source, cross-platform JavaScript runtime environment built on Google Chrome's V8 engine that executes JavaScript code outside the browser using an event-driven, non-blocking input and output model. This architecture enables a single Node.js process to handle thousands of concurrent connections efficiently by delegating I/O operations to the operating system rather than blocking execution, making it well-suited for real-time and data-intensive web applications. In ECOBUD, Node.js serves as the runtime environment for the entire backend API in `apps/api`, executing all request routing, business logic, Prisma-based database access, authentication, scheduled background tasks for presence session cleanup and lesson publishing, and SHA-256 transparency audit logging through its built-in `crypto` module. The developers selected Node.js because its non-blocking I/O model is well-suited for an API that performs frequent concurrent database queries and real-time broadcasts, and its compatibility with TypeScript allows tooling and knowledge to be shared across all project layers. Therefore, the developers chose Node.js as the server-side runtime environment for the backend API of ECOBUD.

---

### Express.js

According to Hahn (2016), Express.js is a minimal and unopinionated web application framework for Node.js that provides utilities for HTTP request routing, middleware composition, and response handling without imposing a fixed application structure. Express.js organizes backend logic by mapping HTTP methods and URL patterns to handler functions and composing middleware that sequentially processes incoming requests, enabling cross-cutting concerns such as authentication, rate limiting, and error handling to be applied consistently across all routes. In ECOBUD, Express version 4.19.2 structures the RESTful API across fifteen registered route modules: `/api/auth`, `/api/users`, `/api/challenges`, `/api/habits`, `/api/lessons`, `/api/events`, `/api/experience`, `/api/transparency`, `/api/realtime`, `/api/moderation`, `/api/admin`, `/api/home`, `/api/learn`, `/api/faqs`, and `/api/user`. Cross-origin resource sharing is managed via the `cors` package version 2.8.5, and request rate limiting is enforced through `express-rate-limit` version 8.3.2. The developers selected Express.js because its minimal design provides the necessary routing and middleware primitives without imposing structural constraints, and its extensive middleware ecosystem reduced the development effort required for common API concerns. Therefore, the developers chose Express.js as the web application framework for the backend API of ECOBUD.

---

### TypeScript

According to Cherny (2019), TypeScript is a statically typed superset of JavaScript developed and maintained by Microsoft that extends the language with optional type annotations, interfaces, enums, and generics, which are verified at compile time before the source code is transpiled to plain JavaScript for execution. TypeScript introduces a compile-time type checking layer that catches type mismatches, null reference errors, and incorrect function signatures before runtime, without introducing any additional runtime overhead. In ECOBUD, TypeScript is the programming language used uniformly across the entire system — version 5.4.5 in `apps/api`, version 6.0.2 in `apps/web`, and version 6.0.3 in `apps/mobile` — enforcing the shapes of request bodies, API response payloads, database record types, React components, and React Native screens across all layers. The `tsx` library version 4.9.3 enables direct execution of TypeScript files in the backend during development without a prior compilation step. The developers selected TypeScript because its static type system provides early detection of errors across a large and layered codebase, and its uniform adoption across all three application layers ensures consistent type contracts throughout the system. Therefore, the developers chose TypeScript as the primary programming language for the full-stack development of ECOBUD.

## 3.X.3 Database Layer Technologies

### PostgreSQL

According to Obe and Hsu (2017), PostgreSQL is an open-source, object-relational database management system that provides full Atomicity, Consistency, Isolation, and Durability (ACID) compliance, support for complex multi-table SQL queries, robust referential integrity enforcement through foreign key constraints, and a rich set of index types. PostgreSQL manages data in structured tables with defined schemas and supports multi-statement transactions that are committed or rolled back atomically, ensuring that complex multi-step operations either complete fully or leave the database unchanged. In ECOBUD, PostgreSQL serves as the primary persistent data store for all relational application data. The Prisma schema defines twenty-three models spanning user identity and access control, gamification and progress tracking, content management, transparency audit logging, community event coordination, swap goods management, and platform administration, hosted on Supabase's managed cloud infrastructure. The developers selected PostgreSQL because its ACID-compliant transaction support and foreign key enforcement are essential for maintaining the integrity of the highly interconnected gamification, habit-tracking, and community engagement data models. Therefore, the developers chose PostgreSQL as the relational database management system for the development of ECOBUD.

---

### Prisma ORM

According to Prisma Data (2023), Prisma is a next-generation open-source Object-Relational Mapper for Node.js and TypeScript that provides a declarative schema language for defining data models, an automated migration engine, and a fully type-safe query builder that auto-generates TypeScript types directly from the database schema. Prisma operates through a schema-first workflow in which all data models, field types, relations, and constraints are defined in a single `schema.prisma` file, from which a typed client library is generated in which every query method, return type, and filter argument is statically typed to match the database structure. In ECOBUD, Prisma with `@prisma/client` version 5.22.0 is used within `apps/api` to define and manage the full database schema, handle all schema migrations through the `prisma migrate` workflow, seed initial data through `prisma/seed.ts`, and provide the typed query interface for all database operations across every service class and route handler. The Prisma datasource block connects to the Supabase-hosted PostgreSQL database via the `DATABASE_URL` and `DIRECT_URL` environment variables. The developers selected Prisma because its auto-generated TypeScript client integrates naturally with the TypeScript-first architecture of the project and its migration workflow provides a clear, auditable history of all schema changes. Therefore, the developers chose Prisma as the Object-Relational Mapper for the development of ECOBUD.

---

### Supabase

According to Richardson (2023), Supabase is an open-source Backend-as-a-Service platform that provides a managed and hosted PostgreSQL database instance, a real-time WebSocket subscription system, a built-in authentication service, and cloud object storage, all accessible through a unified JavaScript client library and a web-based management console. Supabase exposes the underlying PostgreSQL database through a standard connection string, allowing any PostgreSQL-compatible ORM to connect directly without proprietary abstraction, while its managed infrastructure handles server provisioning, automated backups, high availability, and connection pooling. In ECOBUD, Supabase is used as the cloud hosting platform for the PostgreSQL database, with the backend API connecting via the `DATABASE_URL` and `DIRECT_URL` environment variables. The `@supabase/supabase-js` client library version 2.103.0 is used across the backend API and mobile application for client initialization, session management, and Supabase Realtime channel subscriptions. The developers selected Supabase because it eliminates the operational burden of managing a PostgreSQL server while remaining fully compatible with Prisma, and its bundled real-time broadcasting capability enabled live data synchronization without a separate infrastructure component. Therefore, the developers chose Supabase as the cloud database hosting platform for the development of ECOBUD.

## 3.X.4 Third-Party Services

### Supabase Realtime

According to Richardson (2023), Supabase Realtime is a WebSocket-based event broadcasting service built on PostgreSQL's logical replication feature that enables server and client applications to publish messages to named channels and receive them in real time without polling. Supabase Realtime operates through a publish-subscribe model in which clients open a persistent WebSocket connection to the Realtime server and subscribe to named channels, which then deliver broadcast messages instantaneously, eliminating the latency and server load of polling-based approaches to live data delivery. In ECOBUD, Supabase Realtime is implemented through the `SupabaseRealtimeService` class in the backend API, which manages a structured channel architecture: global channels (`ecobud:global:challenges`, `ecobud:global:learn`) for content refresh signals, HMAC-SHA256 hashed per-user channels for personalized notices and refresh signals, administrator channels (`ecobud:admin:dashboard`, `ecobud:admin:users`) for dashboard updates, and a presence channel (`ecobud:presence:members`) for tracking live session activity displayed on the administrator dashboard. The developers selected Supabase Realtime because its integration within the existing Supabase platform eliminated the need to provision a separate WebSocket server, and its channel-based publish-subscribe model mapped naturally to the domain-specific update requirements of both the mobile application and the administrator dashboard. Therefore, the developers chose Supabase Realtime as the real-time event delivery service for the live data synchronization features of ECOBUD.

## 3.X.5 Development Environment and Tooling

### Visual Studio Code

According to Mullen (2016), Visual Studio Code is a free, open-source code editor developed by Microsoft that provides syntax highlighting, intelligent code completion through the Language Server Protocol, integrated debugging, source control integration, and an extensible marketplace of language and tool extensions supporting virtually all major programming languages and frameworks. Visual Studio Code communicates with language servers running in the background to deliver real-time type-checking feedback, symbol navigation, and code auto-completion without the overhead of a full IDE, while its extension marketplace consolidates language support, linters, and platform toolchains within a single editor interface. In ECOBUD, Visual Studio Code served as the primary development environment across all application layers. The TypeScript Language Server provided real-time diagnostics and refactoring assistance across the entire monorepo, the Prisma extension provided schema syntax highlighting and auto-completion, the Tailwind CSS IntelliSense extension provided utility class preview, and ESLint together with Prettier enforced consistent code quality and formatting standards across all three application packages. The developers selected Visual Studio Code because its unified support for TypeScript, React, React Native, and Prisma through the extension marketplace allowed all layers of the full-stack project to be developed and debugged within a single editor, significantly reducing context switching. Therefore, the developers chose Visual Studio Code as the primary Integrated Development Environment for the development of ECOBUD.

---

### Android Studio

According to Leiva (2015), Android Studio is the official Integrated Development Environment for Android application development, built on the IntelliJ IDEA platform and distributed by Google. It provides the Android SDK Manager for installing SDK platform versions and build tools, the Android Virtual Device Manager for creating and managing emulated devices, and a suite of performance profiling utilities. Android Studio provides the underlying Android development toolchain that cross-platform frameworks such as React Native depend upon, including the SDK, platform tools, and system images required to compile and simulate Android targets. In the development of ECOBUD, Android Studio was used to configure the Android development environment required for testing the Expo-based mobile application. The SDK Manager was used to install the required SDK platform versions and build tools, while the AVD Manager was used to create and launch emulated Android devices for local testing through the Expo development server. The developers selected Android Studio because the Android SDK Manager and AVD Manager it provides are required dependencies of the React Native and Expo toolchain, making its use a necessary prerequisite for local Android device testing. Therefore, the developers chose Android Studio as the Android platform management tool for the development of ECOBUD.

---

### Git

According to Chacon and Straub (2014), Git is a free and open-source distributed version control system designed to handle projects of any size with speed and efficiency. It records snapshots of project files over time, supports concurrent development through isolated branches, and provides tools for merging divergent lines of work back into a unified history. Git maintains a directed acyclic graph of commits, each recording the complete state of tracked files along with a reference to its parent, enabling parallel feature development, safe experimentation, and complete rollback to any prior project state. In ECOBUD, Git was used as the version control system throughout the entire development lifecycle. The monorepo structure containing `apps/mobile`, `apps/web`, and `apps/api` was managed within a single Git repository, allowing cross-layer changes to be committed atomically and the complete version history of all three packages to be tracked together. The developers selected Git because its distributed branching model supports parallel feature development across multiple application layers while maintaining a stable integration branch, and its widespread adoption ensures compatibility with the collaborative hosting tools used in the project. Therefore, the developers chose Git as the version control system for the development of ECOBUD.

---

### GitHub

According to Beer (2018), GitHub is a cloud-based hosting platform for Git repositories that augments the core Git version control system with web-based collaboration features including pull request review workflows, issue tracking, project management boards, and integration hooks for continuous integration and deployment pipelines. GitHub provides a centralized, network-accessible remote location for storing Git repositories, enabling distributed team collaboration through pull requests for code review and an issue tracker for structured task management. In ECOBUD, GitHub was used as the remote repository hosting platform, providing centralized and persistent storage of the complete version history, a web interface for browsing code and commits, and collaborative infrastructure for coordinating development work across the team. The developers selected GitHub because it is the most widely adopted platform for hosting Git repositories and provides the collaboration features necessary for coordinated multi-developer software projects. Therefore, the developers chose GitHub as the remote repository hosting platform for the development of ECOBUD.
